Sprint Manager

The Sprint Manager combines two tools: the Sprint Creator for adding new sprints to Azure DevOps, and the Sprint Populator for filling sprints with template work items. Switch between tabs at the top of the page.

Sprint Creator

Create new iterations (sprints) in Azure DevOps directly from the dashboard without navigating to the Azure DevOps admin settings.

How to Use It

  1. Open Sprint Manager from the sidebar and select the Sprint Creator tab.
  2. Select Project & Team — Choose the project and team for the new sprint.
  3. Review the Existing Sprints table to see what already exists.
  4. Sprint Details — The system auto-suggests a name, parent path, and dates based on existing sprint patterns. Adjust as needed.
  5. Click Create Sprint to create the iteration only, or Create & Add to Team to also assign it to the selected team immediately.

Auto-Inference

The Sprint Creator analyses your existing sprints and automatically suggests:

  • The next sprint name (based on naming patterns like "Sprint 15" → "Sprint 16")
  • The parent path in the iteration hierarchy
  • Start and end dates (continuing from the last sprint's end date)
Tip After creating a sprint, click Create another to immediately set up the next one — the auto-inference updates automatically.

Sprint Populator

The Sprint Populator helps you set up a new sprint by automatically copying template user stories and their tasks. If your team has recurring sprint tasks (like "Sprint Planning", "Retrospective", or "Deploy to Production"), you only need to create them once as templates.

Previewing sprint templates before cloning into a new sprint.

How Templates Work

A template is simply a User Story in Azure DevOps that has the tag sprint template. All child tasks under that story are included as part of the template.

When you use the Sprint Populator, it clones these template stories (and their tasks) into your target sprint.

You can optionally select a different source project for template lookup. This is useful when one project acts as your template library for multiple delivery projects.

Automatic Title Replacement

If a template title contains the letter X as a standalone word, it is replaced with the target sprint number. For example:

  • "Sprint X Planning" becomes "Sprint 15 Planning" (if your target is Sprint 15).
  • "Deploy Release X" becomes "Deploy Release 15".

How to Use It

  1. Select the Sprint Populator tab.
  2. Step 1 — Select your source project (where templates live) and target project (where new work items are created). These can be the same project.
  3. Step 2 — Select the target team.
  4. Step 3 — Select the target sprint. The sprint dates are shown to help you pick the right one.
  5. Step 4 — Preview which templates will be cloned. Use the checkboxes to include or exclude individual stories and tasks. Click Apply.

What Gets Created

  • A new User Story for each selected template, placed in the target sprint.
  • All selected child Tasks from the template are cloned under the new story.
  • The sprint template tag is removed from cloned items so they are not treated as templates.

Setting Up Templates

To create a sprint template in Azure DevOps:

  1. Create a User Story with the tasks you want to repeat every sprint.
  2. Add the tag sprint template to the User Story.
  3. That is it — the Sprint Populator will find it automatically.
Tip You can have multiple templates. Each one is cloned independently when you run the populator.
Important The Sprint Populator creates real work items in Azure DevOps. Double-check the preview before clicking Apply.
