Settings

The Settings panel lets you manage your connections and security. Open it by clicking the gear icon (⚙) in the top-right corner of any screen.

The Settings modal.

Tabs

Settings are organised into three tabs: General, Database, and Email.

General Tab

Demo Mode

When enabled, all names, project titles, and other identifying information are anonymised. This is useful when presenting the dashboard to an audience without revealing real data. Toggle it on or off with the switch.

Password (API Key)

This is the password you use to log in to the dashboard. You can change it here at any time. After changing it, you will need to use the new password the next time you log in.

Passwords are encrypted at rest using Windows DPAPI with an application-specific salt, bound to your user account.

Azure DevOps Connection (PAT)

Your Personal Access Token allows DevOps InControl to read data from Azure DevOps. A green PAT OK indicator in the header bar means the connection is active.

If you need to update or rotate your PAT:

  1. Create a new PAT in Azure DevOps (see First-Time Setup for instructions).
  2. Paste it into the PAT field in Settings.
  3. Click Save.
Important If your PAT expires, checks will stop working and you will see a red PAT Missing indicator. Replace the PAT to resume.

Database Tab

Database Servers

DevOps InControl can connect to up to three database servers for the DB Monitor feature. For each server you can configure:

FieldDescription
Server addressThe hostname or IP address of the database server.
PortThe connection port (default: 1433 for SQL Server, 5432 for PostgreSQL).
UsernameA database user with read access.
PasswordThe password for that user.
DriverChoose SQL Server or PostgreSQL.

After entering the details, click Test Connection to verify that DevOps InControl can reach the server.

Email Tab

Email Sender

Some features (like sending reminders for missing estimates) can send emails. Enter the "From" email address here. The mail server (SMTP) is configured by your administrator.

Tip All credentials (password, PAT, database passwords) are stored securely on the server using Windows DPAPI encryption with an application-specific salt. They are never sent back to your browser in plain text.
